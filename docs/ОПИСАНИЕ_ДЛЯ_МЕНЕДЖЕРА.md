# Convvy — функционал и архитектура (кратко)

**Convvy** — веб-шпаргалка для операторов call-центра: во время звонка оператор
за пару секунд находит готовый ответ на нужном языке. Наполнение контента ведёт
менеджер через отдельную админ-панель, без программиста.

---

## Функционал

**Приложение оператора**
- Вход по одноразовому коду из письма (только для email из разрешённого списка).
- Выбор языка: русский, польский, немецкий, испанский, английский.
- 4 раздела на главном экране:
  - **Возражения** — пошагово: возражение → этап разговора → готовый ответ
    (+ подсказки «а если клиент скажет…»).
  - **Презентации**, **Сервисы**, **Рынок** — списки готовых материалов.
- Копирование текста одной кнопкой.
- Показывается только тот контент, что реально переведён на выбранный язык.

**Админ-панель** (операторы её не видят)
- Добавление/редактирование/удаление: возражения, этапы, ответы, презентации,
  сервисы, материалы рынка.
- Мультиязычность: сначала выбирается язык, затем заполняется поле под него;
  один материал — несколько переводов.
- Управление порядком (стрелки ▲/▼), режим «черновик» (не виден операторам).
- Управление списком языков и списком операторов, которым разрешён вход.

---

## Архитектура (из чего это состоит)

| Компонент | Что это | За что отвечает |
|-----------|---------|-----------------|
| **Интерфейс (фронтенд)** | Веб-приложение в браузере (React) | Всё, что видит оператор и менеджер; работает на ПК и телефоне |
| **База данных** | Supabase (PostgreSQL) | Хранит весь контент, языки, списки операторов и админов |
| **Вход / авторизация** | Supabase Auth | Коды входа для операторов, пароль для админов, проверка прав |
| **Почта** | SMTP (Gmail) через Supabase | Отправка одноразовых кодов входа на email |
| **Хостинг** | Netlify | Публикует сайт в интернете, автообновление при изменениях |
| **Хранилище кода** | GitHub | Где хранится и версионируется код; из него собирается сайт |
| **Автоподдержка БД** | GitHub Actions (по расписанию) | Не даёт бесплатной базе «уснуть» от простоя |

**Как это связано (простыми словами):**
браузер (интерфейс) ↔ база данных Supabase (контент + вход) → письма с кодами
через почту; сайт размещён на Netlify и автоматически пересобирается из GitHub
при каждом изменении.

**Защита данных:** контент общедоступен только для чтения; изменять что-либо
может лишь администратор. Служебные списки (операторы, админы) закрыты от
посторонних на уровне базы данных.
